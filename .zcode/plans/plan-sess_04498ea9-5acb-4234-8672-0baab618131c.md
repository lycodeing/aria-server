# Knowledge-Client 改造方案（参照 Auth-Client）

## 目标
将 `knowledge-client` 模块改造为与 `auth-client` 一致的架构风格，修复 `R<T>` 响应反序列化 Bug，补齐异常处理、校验、自动装配等能力。

## 文件变更清单

### 1. 新增文件

| # | 文件 | 说明 |
|---|------|------|
| 1 | `knowledge-client/src/main/java/com/aria/sdk/knowledge/internal/ApiResponse.java` | 内部 `R<T>` 响应包装映射（对照 `auth/internal/ApiResponse.java`），`record` 类型，字段 `code/msg/data/traceId` |
| 2 | `knowledge-client/src/main/java/com/aria/sdk/knowledge/exception/KnowledgeClientException.java` | 业务异常类，继承 `SdkException`，携带 `httpStatus`/`bizCode`（对照 `AuthClientException`） |
| 3 | `knowledge-client/src/main/java/com/aria/sdk/knowledge/KnowledgeClientProperties.java` | `@ConfigurationProperties(prefix="knowledge.client")` 配置类，含 `@Validated` fail-fast（对照 `AuthClientProperties`） |
| 4 | `knowledge-client/src/main/resources/META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports` | Spring Boot 3.x 自动装配注册文件 |

## 核心改造细节

### `KnowledgeClient.java` 核心变化

```java
// 之前（有 Bug）
public SearchResponse search(SearchRequest request) {
    return post("/internal/knowledge/search", request, SearchResponse.class);  // ❌ R<T> 未解包
}

// 之后
public SearchResponse search(SearchRequest request) {
    if (request == null) {
        throw new IllegalArgumentException("SearchRequest 不能为空");
    }
    ApiResponse<SearchResponse> resp = doPost(
        "/internal/knowledge/search", request,
        new TypeRef<ApiResponse<SearchResponse>>() {},
        "知识库检索失败");
    return unwrap(resp, "知识库检索失败");
}
```

### 异常处理链

```
SdkException (HTTP层)
  → wrapHttpFailure() → KnowledgeClientException(httpStatus, bizCode=-1)

ApiResponse.code != 200 (业务层)
  → unwrap() → KnowledgeClientException(httpStatus=-1, bizCode=实际code)
```

消费方可以基于 `httpStatus`/`bizCode` 做细粒度熔断。

## 不变更项
- DTO 类（`SearchRequest`/`SearchResponse`/`ChunkHitDTO`）结构不变，服务端返回的 `Map<String,Object>` 能与现有 DTO 正常反序列化
- `BaseClient` 公共基础设施不动
